# 人事・労務の行政公式サイト検索スキル (hr-search)

## 概要

人事・労務に関するユーザーの自然文の質問を、日本の行政・公的機関の公式サイトのみに絞って検索し、結果を整理してわかりやすく回答する。

## ワークフロー

### ステップ1: サイト管理コマンドの確認

ユーザーのメッセージが以下のいずれかの場合、検索ではなく管理操作を行う：

| ユーザーの発言例 | 操作 |
|---|---|
| 「サイト一覧を見せて」「対象サイトを表示」 | → **[サイト一覧表示]** を実行 |
| 「〇〇.go.jpを追加して」 | → **[サイト追加]** を実行 |
| 「〇〇.go.jpを削除して」「〇〇.go.jpを外して」 | → **[サイト削除]** を実行 |
| 「サイトをリセットして」「デフォルトに戻して」 | → **[リセット]** を実行 |

#### [サイト一覧表示]
現在の検索対象サイト一覧をわかりやすく表示する。初回または変更がなければデフォルトを表示。

#### [サイト追加]
指定されたドメインを一覧に追加し、追加後の一覧を表示する。

#### [サイト削除]
指定されたドメインを一覧から削除し、削除後の一覧を表示する。

#### [リセット]
サイト一覧をデフォルトに戻す。

**重要:** サイト一覧の変更は **会話スコープ内でのみ有効**。Claudeはステートレスなので、会話をまたいだ永続的な保存はできない旨をユーザーに伝える。

### ステップ1.5: 質問の曖昧さチェック（事例ベースの質問の場合）

ユーザーの質問が **具体的な事例（人物・状況・数値が含まれる相談形式）** の場合、検索を実行する前に、検索の精度を左右する論点について確認質問を行う。

#### 確認が必要なケース

以下のいずれかに該当する場合は、検索前にユーザーに確認する：

1. **原則論と例外論のどちらを知りたいかが質問から判断できない**
   - 例：加入要件の原則 vs 遡及適用・特例措置・実態優先などの例外運用

2. **判断を分ける数値情報（期間・継続性・頻度）について、ユーザーの記述に幅がある**
   - 例：「残業が増えてきた」「最近〜することが多い」「ときどき〜」
   - これらは「一時的か恒常的か」で行政の取扱いが分かれるため、確認すべき

3. **行政運用の論点が複数ありうる**
   - 例：契約変更の要否／遡及適用の有無／罰則・遡及請求リスク／実務手続きなど

4. **ユーザーが知りたい立場が不明**
   - 例：事業主としてのリスク回避なのか、労働者としての権利確認なのか

#### 確認が不要なケース（即検索）

以下は曖昧さがほぼないため、確認なしで検索に進む：

- 制度・料率・期日の単純確認（例：「令和8年度の健康保険料率は？」）
- 書式・手続きの所在確認（例：「離職票の記入例はどこにある？」）
- 法令条文の確認（例：「労基法第36条の内容を教えて」）
- 用語の定義（例：「みなし労働時間制とは？」）

#### 確認質問の形式

`ask_user_input_v0` ツールを使い、論点を選択肢として提示する。

**選択肢の最初は必ず「Claudeに任せる」を入れる。** これにより、ユーザーは「手間をかけて曖昧さをなくす」か「手間をかけずにClaudeに解釈を委ねる」かを能動的に選べる。Claudeに任せた結果がイマイチだった場合は、ユーザーが質問を言い換えて再度試すことができる。

**確認質問のテンプレート：**

> ご質問について、どの論点を中心に調べるか教えてください。
> 
> 0. **Claudeに任せる**（質問の文脈から私が論点を解釈して進めます。回答が意図と違っていたら、別の聞き方で再質問してください）
> 1. （論点A）
> 2. （論点B）
> 3. （論点C）
> 4. （論点D）
> 5. すべて知りたい

**選択肢の構成ルール：**
- 「Claudeに任せる」は必ず**先頭（選択肢0または1番目）**に置く
- それ以外の論点は **2〜4個** に絞る（多すぎるとユーザーの負担になる）
- 最後に「すべて知りたい」を追加（網羅的に知りたいユーザー向け）
- 各論点は具体的に書く（「原則論」だけでなく「加入要件の原則論（所定労働時間 vs 実労働時間の判断基準）」のように、何を調べるかが伝わる粒度にする）

#### 「Claudeに任せる」が選ばれた場合の動作

1. 質問の文脈から最も中心的と思われる論点を1〜2個推定する
2. ただし、原則論だけでなく、**例外論・運用基準にもキーワードを広げて検索する**（例：「遡及」「実態」「継続」などのキーワードを念のため加える）
3. 回答の冒頭で「ご質問の文脈から〜を中心に調べました」と一言添える
4. 回答の末尾に以下のメッセージを表示：

   > 💡 質問の意図を私が解釈して回答しました。もし論点が違っていたら、別の聞き方（例：「遡及適用について」「罰則について」など）で再度ご質問ください。

#### 確認をスキップしたい場合

ユーザーの最初の質問内に以下のような表現がある場合、確認をスキップして「Claudeに任せる」モードで検索に進む：

- 「ざっくり教えて」「概要を知りたい」「とりあえず」
- 「すぐに回答してほしい」「確認は不要」「任せる」

#### キーワード抽出への反映

ユーザーの回答（選択された論点）に応じて、ステップ2のキーワード抽出に**論点固有のキーワード**を加える：

| 論点 | 追加すべきキーワード例 |
|---|---|
| 遡及適用 | 「遡及」「資格取得日」「概ね6か月」「継続」「乖離」 |
| 実態優先 | 「雇用実態」「実態」「乖離」「所定外労働」 |
| リスク・罰則 | 「罰則」「遡及請求」「未加入」「確認請求」 |
| 実務手続き | 「届出」「変更届」「タイミング」 |

これにより、原則論を解説する一般ページではなく、行政運用の例外論を扱うリーフレットや通達PDFにヒットしやすくなる。

### ステップ2: 検索URLの構築

ユーザーの自然文の質問から検索キーワードを抽出し、各サイトの検索URLを構築する。

1. **質問からキーワードを抽出**（5〜10語程度）
2. **各サイトのサイト内検索URLを構築する**

サイトごとの検索URL形式：

| サイト | 検索URL形式 |
|---|---|
| mhlw.go.jp | `https://www.mhlw.go.jp/stf/seisakunitsuite/bunya/koyou_roudou/index.html`（トップ）または `https://www.mhlw.go.jp/search/?q=キーワード` |
| e-gov.go.jp | `https://elaws.e-gov.go.jp/search/elawsSearch/elaws_search/lsg0100/` または `https://www.e-gov.go.jp/search/?q=キーワード` |
| nenkin.go.jp | `https://www.nenkin.go.jp/search/search.html?q=キーワード` |
| kyoukaikenpo.or.jp | `https://www.kyoukaikenpo.or.jp/search/?q=キーワード` |
| courts.go.jp | `https://www.courts.go.jp/search/?q=キーワード` |

**汎用パターン（上記で取得できない場合）：**
各サイトのトップページやFAQページを直接 `web_fetch` で取得し、関連リンクを探してさらに取得する。

### ステップ3: コンテンツの取得

#### フェーズA：行政公式サイト限定検索（必須）

`web_fetch` ツールを使い、**現在のサイトリストに含まれるサイトのみ**からコンテンツを取得する。

**手順：**
1. 質問に最も関連しそうなサイト（1〜3件）を優先して `web_fetch` でページを取得
2. ページ内に関連リンクがあれば、さらに `web_fetch` で深掘り取得（最大3ページ程度）
3. 取得できた情報が不十分な場合、別のサイトリスト内のサイトも試みる
4. **サイトリスト外のサイトから取得したコンテンツは回答に使用しない**

**重要：** フェーズAでは以下のルールを厳守する。
- `web_search` を使用する場合、返ってきた検索結果のURLをすべて使用するのではなく、**サイトリスト内のドメインのURLのみを抽出**して `web_fetch` する。
- サイトリスト外のドメインのURLは `web_fetch` しない。検索結果スニペットに含まれる情報も回答に使用しない。
- フェーズAで意図せずサイトリスト外の情報を参照した場合は、後述の⚠️表示ルールに従い明示する。

#### フェーズB：検索範囲拡大（条件付き）

フェーズAの結果が以下のいずれかに該当する場合、回答を表示する前に必ずユーザーに確認する：
- 取得したページに質問に直接答える記述が見当たらない
- 情報が断片的で、質問に対して十分な回答が構成できない

**確認メッセージの形式：**

> 「（具体的な理由）ため、対象となる行政公式サイトからは十分な回答が見つかりませんでした。対象となる行政公式サイト以外も検索しますか？」

ユーザーが「検索する」「はい」「お願いします」など、範囲拡大に同意した場合のみ：
1. `web_search` ツールを使い、サイトリスト外も含めた通常の検索を実施する
2. 取得した情報を使って回答を構成する
3. **対象となる行政公式サイト以外の情報であることを明示する**（後述の回答形式参照）
4. サイトリスト外の参考URLは通常のURLと区別して表示する（後述参照）

ユーザーが拒否した場合は、行政公式サイトで見つかった範囲の情報のみを回答する（情報が不十分である旨を明記）。

### ステップ4: 回答の構成

検索結果をもとに、以下の形式でまとめる：

```
## 回答

[質問に対する結論を最初に明確に述べる]

### 根拠・詳細

[取得した公式情報を箇条書きや段落で整理]

### 参考URL

- [ページタイトル](完全なURL)（サイト名）📄 ※PDFの場合は末尾に「📄 PDF」と明記
- ...
```

**【サイトリスト外情報を含む場合の回答形式】**

以下のいずれかに該当する場合、回答内でその情報を明示する：
- フェーズBでサイトリスト外も検索した場合
- フェーズAでサイトリスト外の情報が意図せず混入した場合（検索結果スニペット等）

```
## 回答

[質問に対する結論を最初に明確に述べる]

### 根拠・詳細（行政公式サイト）

[行政公式サイトで取得できた情報があれば記載]

### 根拠・詳細（⚠️ 対象となる行政公式サイト**以外**の情報）

[行政公式サイト以外から取得した情報を記載]

### 参考URL

#### 行政公式サイト
- [ページタイトル](完全なURL)（サイト名）

#### ⚠️ 対象となる行政公式サイト**以外**
- [ページタイトル](完全なURL)（サイト名・運営者） 🔗 ※対象となる行政公式サイト**以外**

> ⚠️ 対象となる行政公式サイト**以外**の情報が含まれています。
```

**【正確性の基準】**

以下の場合は回答を表示せず、その旨をユーザーに伝える（ステップ3フェーズBの確認に移行）：
- 取得したページに質問に直接答える記述が見当たらない
- 複数ページの情報が矛盾しており、どちらが正しいか判断できない
- 法改正等により情報が古い可能性が高く、現行ルールが不明確

曖昧・不確実な情報は回答に含めない。確認できた事実のみを記述する。
「〜と思われます」「〜かもしれません」などの推測表現は使用しない。
法令・通達の条文番号が確認できた場合は必ず記載する。

**【参考URLの表示ルール】**

- 回答に使用したすべてのページの完全なURLを「参考URL」セクションに列挙する
- HTMLページとPDFが両方含まれる場合は種別を必ず区別して表示する

**【PDF URLの有効性確認ルール】**

参考URLにPDF（`.pdf` で終わるURL）が含まれる場合、ユーザーに表示する前に以下の手順で有効性を確認する：

1. `web_fetch` で各PDF URLにアクセスし、正常に取得できるか確認する（`text_content_token_limit=10` で最小限のfetchでよい）
2. 確認対象は **PDFのみ**（HTMLページは対象外）
3. 確認件数は **最大5件まで**（5件を超える場合は先着5件のPDFのみ確認し、残りはそのまま表示）
4. 結果に応じて以下のように表示する：
   - **正常に取得できた場合** → そのURLをそのまま表示する
   - **404等で取得できなかった場合** → そのPDF URLは表示せず、代わりにそのサイトの関連一覧ページまたはトップページのURLを表示し、末尾に「※元のURLが無効のため一覧ページを表示」と注記する

PDFを参考にした場合は、以下の形式で表示する：

```
- [ドキュメントタイトル](完全なURL) 📄 PDF（サイト名）
  - 該当ページ：X ページ / 全 N ページ
  - 根拠箇所：「〜〜〜（該当する文言・単語・文章をそのまま引用）」
```

**PDFページ情報の取得方法：**
- `web_fetch` でPDFを取得した際、ページ数情報（総ページ数・該当ページ番号）を抽出する
- 根拠に使った具体的な文言（単語・フレーズ・文章）をそのまま引用して記載する
- 該当箇所が複数ページにわたる場合は `X〜Y ページ / 全 N ページ` と範囲で示す
- ページ数が取得できない場合は「ページ数不明」と記載する

**【PDF限定再検索オファー】**

参考URLにHTMLページとPDFが**混在**している場合、回答の最後に必ず以下のメッセージを表示する：

> 📎 回答にHTMLページとPDFの両方が含まれています。
> **「PDFに限定して再検索する」** と入力すると、PDFのみを対象に同じ質問で再検索します。

ユーザーが「PDFに限定して再検索する」または同趣旨の発言をした場合：
1. 直前の質問と同じキーワードで再度 `web_fetch` を実行
2. 取得URLが `.pdf` で終わるページのコンテンツのみを使用して回答を構成する
3. HTMLページの情報は使用しない
4. 参考URLはPDFのみを列挙する

## 利用例

### 例1: 事例ベースの質問（確認プロセスあり）

**ユーザー:** 「雇い入れ当初週18時間の社員が残業で20時間を超えるようになった場合、雇用保険に加入すべきですか？」

**Claude（ステップ1.5の確認）:**
> ご質問について、どの論点を中心に調べるか教えてください。
> 
> 0. **Claudeに任せる**（質問の文脈から私が論点を解釈して進めます。回答が意図と違っていたら、別の聞き方で再質問してください）
> 1. 加入要件の原則論（所定労働時間 vs 実労働時間の判断基準）
> 2. 遡及適用の有無・期間基準（実態が継続した場合の扱い）
> 3. 実務対応（雇用契約書の変更、ハローワークへの届出）
> 4. 未加入のリスク（罰則、労働者からの遡及請求）
> 5. すべて知りたい

→ ユーザーが「2と3」を選択 → 「遡及」「資格取得日」「6か月」「実態」「届出」をキーワードに加えて検索 → 結果まとめ → 回答表示

---

## スキルの使い方説明

ユーザーが「このスキルの使い方を知りたい」「このスキルをどうやって使うの？」など、スキルの使い方を尋ねた場合、以下の内容を説明する。

---

**このスキル（hr-search）でできること：**

人事・労務に関する行政・公的機関の公式サイトだけを情報源にして、あなたの質問に答えます。

**メリット：**
- 厚生労働省・年金機構・e-Govなど信頼性の高い公式情報のみを参照するため、民間サイトや個人ブログなどの不正確な情報に惑わされない
- 雇用保険・社会保険・労働基準・育休・有給・就業規則など、人事・労務の実務に関する質問を自然な文章で聞ける
- 複数の省庁サイトをまとめて調べ、結論を分かりやすく整理して回答する
- 具体的な事例の質問では、検索前に論点を確認して精度を高めることができる（「Claudeに任せる」も選択可）

**使い方：**
普通に質問するだけでOKです。例えば：
- 「育児休業給付金はいつまでもらえますか？」
- 「パートの社会保険加入条件を教えてください」
- 「時間外労働の上限規制はどうなっていますか？」
- 「試用期間中の解雇は認められますか？」

事例ベースの質問（具体的な状況や数値を含むもの）では、論点を確認する選択肢が表示されることがあります。手間をかけずにすぐ回答が欲しい場合は「Claudeに任せる」を選んでください。

**対象サイトの管理：**
- 「対象サイトの一覧を見せて」→ 現在の検索対象サイト一覧を表示
- 「〇〇.go.jp を追加して」→ サイトを追加
- 「〇〇.go.jp を削除して」→ サイトを除外
- 「デフォルトに戻して」→ 初期設定にリセット

---

## 注意事項

- 検索結果が行政サイト以外のページを含む場合、その情報は無視して行政サイトの情報のみを使う（フェーズBで範囲拡大した場合を除く）
- 質問の内容が複数の省庁にまたがる場合（例：雇用保険＋年金）、複数のサイトを横断して検索・統合する
- 法改正がある分野（育児休業、最低賃金など）は情報の日付に注意し、最新情報を優先する
- **事例ベースの質問では、検索前にステップ1.5の曖昧さチェックを実施すること。確認質問の選択肢には必ず「Claudeに任せる」を最初に含め、ユーザーが手間をかけて論点を絞るか、Claudeに解釈を委ねるかを能動的に選べるようにする**
- **「Claudeに任せる」が選ばれた、または質問内にスキップ表現があった場合は、原則論だけでなく例外論・運用基準にもキーワードを広げて検索すること**
- **回答の末尾に「最終的には所管機関にご確認ください」「○○機関にお問い合わせください」など、確認・問い合わせを促す定型メッセージを付加しない**
- **回答の末尾に「このスキルは行政サイトを対象にしています」「行政公式サイトを検索しました」など、スキルの動作説明や情報源の種類を説明するメッセージを付加しない**
