# 人事・労務の行政公式サイト検索スキル (hr-search)

## 概要

人事・労務に関するユーザーの自然文の質問を、日本の行政・公的機関の公式サイトのみに絞って検索し、結果を整理してわかりやすく回答する。

## ワークフロー

### ステップ1: サイト管理コマンドの確認

ユーザーのメッセージが以下のいずれかの場合、検索ではなく管理操作を行う：

| ユーザーの発言例 | 操作 |
|---|---|
| 「サイト一覧を見せて」「対象サイトを表示」 | → **[サイト一覧表示]** を実行 |
| 「〇〇.go.jpを追加して」 | → **[サイト追加]** を実行 |
| 「〇〇.go.jpを削除して」「〇〇.go.jpを外して」 | → **[サイト削除]** を実行 |
| 「サイトをリセットして」「デフォルトに戻して」 | → **[リセット]** を実行 |

#### [サイト一覧表示]
現在の検索対象サイト一覧をわかりやすく表示する。初回または変更がなければデフォルトを表示。

#### [サイト追加]
指定されたドメインを一覧に追加し、追加後の一覧を表示する。

#### [サイト削除]
指定されたドメインを一覧から削除し、削除後の一覧を表示する。

#### [リセット]
サイト一覧をデフォルトに戻す。

**重要:** サイト一覧の変更は **会話スコープ内でのみ有効**。Claudeはステートレスなので、会話をまたいだ永続的な保存はできない旨をユーザーに伝える。

---

### ステップ2: 検索URLの構築

ユーザーの自然文の質問から検索キーワードを抽出し、各サイトの検索URLを構築する。

1. **質問からキーワードを抽出**（5〜10語程度）
2. **各サイトのサイト内検索URLを構築する**

サイトごとの検索URL形式：

| サイト | 検索URL形式 |
|---|---|
| mhlw.go.jp | `https://www.mhlw.go.jp/stf/seisakunitsuite/bunya/koyou_roudou/index.html`（トップ）または `https://www.mhlw.go.jp/search/?q=キーワード` |
| e-gov.go.jp | `https://elaws.e-gov.go.jp/search/elawsSearch/elaws_search/lsg0100/` または `https://www.e-gov.go.jp/search/?q=キーワード` |
| nenkin.go.jp | `https://www.nenkin.go.jp/search/search.html?q=キーワード` |
| kyoukaikenpo.or.jp | `https://www.kyoukaikenpo.or.jp/search/?q=キーワード` |
| courts.go.jp | `https://www.courts.go.jp/search/?q=キーワード` |

**汎用パターン（上記で取得できない場合）：**
各サイトのトップページやFAQページを直接 `web_fetch` で取得し、関連リンクを探してさらに取得する。

---

### ステップ3: コンテンツの取得

#### フェーズA：行政公式サイト限定検索（必須）

`web_fetch` ツールを使い、**現在のサイトリストに含まれるサイトのみ**からコンテンツを取得する。

**手順：**
1. 質問に最も関連しそうなサイト（1〜3件）を優先して `web_fetch` でページを取得
2. ページ内に関連リンクがあれば、さらに `web_fetch` で深掘り取得（最大3ページ程度）
3. 取得できた情報が不十分な場合、別のサイトリスト内のサイトも試みる
4. **サイトリスト外のサイトから取得したコンテンツは回答に使用しない**

**重要：** フェーズAでは `web_fetch` はサイトリストに含まれるドメインのURLのみに使用する。リスト外のURLは取得しない。

#### フェーズB：検索範囲拡大（条件付き）

フェーズAの結果が以下のいずれかに該当する場合、回答を表示する前に必ずユーザーに確認する：
- 取得したページに質問に直接答える記述が見当たらない
- 情報が断片的で、質問に対して十分な回答が構成できない

**確認メッセージの形式：**

> 「（具体的な理由）ため、対象となる行政公式サイトからは十分な回答が見つかりませんでした。対象となる行政公式サイト以外も検索しますか？」

ユーザーが「検索する」「はい」「お願いします」など、範囲拡大に同意した場合のみ：
1. `web_search` ツールを使い、サイトリスト外も含めた通常の検索を実施する
2. 取得した情報を使って回答を構成する
3. **対象となる行政公式サイト以外の情報であることを明示する**（後述の回答形式参照）
4. サイトリスト外の参考URLは通常のURLと区別して表示する（後述参照）

ユーザーが拒否した場合は、行政公式サイトで見つかった範囲の情報のみを回答する（情報が不十分である旨を明記）。

---

### ステップ4: 回答の構成

検索結果をもとに、以下の形式でまとめる：

```
## 回答

[質問に対する結論を最初に明確に述べる]

### 根拠・詳細

[取得した公式情報を箇条書きや段落で整理]

### 参考URL

- [ページタイトル](完全なURL)（サイト名）📄 ※PDFの場合は末尾に「📄 PDF」と明記
- ...
```

**【サイトリスト外情報を含む場合の回答形式】**

フェーズBでサイトリスト外も検索した場合、回答内でその情報を明示する：

```
## 回答

[質問に対する結論を最初に明確に述べる]

### 根拠・詳細（行政公式サイト）

[行政公式サイトで取得できた情報があれば記載]

### 根拠・詳細（⚠️ 対象となる行政公式サイト**以外**の情報）

[行政公式サイト以外から取得した情報を記載]

### 参考URL

#### 行政公式サイト
- [ページタイトル](完全なURL)（サイト名）

#### ⚠️ 対象となる行政公式サイト**以外**
- [ページタイトル](完全なURL)（サイト名・運営者） 🔗 ※対象となる行政公式サイト**以外**

> ⚠️ 対象となる行政公式サイト**以外**の情報が含まれています。
```

**【正確性の基準】**

以下の場合は回答を表示せず、その旨をユーザーに伝える（ステップ3フェーズBの確認に移行）：
- 取得したページに質問に直接答える記述が見当たらない
- 複数ページの情報が矛盾しており、どちらが正しいか判断できない
- 法改正等により情報が古い可能性が高く、現行ルールが不明確

曖昧・不確実な情報は回答に含めない。確認できた事実のみを記述する。
「〜と思われます」「〜かもしれません」などの推測表現は使用しない。
法令・通達の条文番号が確認できた場合は必ず記載する。

**【参考URLの表示ルール】**

- 回答に使用したすべてのページの完全なURLを「参考URL」セクションに列挙する
- HTMLページとPDFが両方含まれる場合は種別を必ず区別して表示する

PDFを参考にした場合は、以下の形式で表示する：

```
- [ドキュメントタイトル](完全なURL) 📄 PDF（サイト名）
  - 該当ページ：X ページ / 全 N ページ
  - 根拠箇所：「〜〜〜（該当する文言・単語・文章をそのまま引用）」
```

**PDFページ情報の取得方法：**
- `web_fetch` でPDFを取得した際、ページ数情報（総ページ数・該当ページ番号）を抽出する
- 根拠に使った具体的な文言（単語・フレーズ・文章）をそのまま引用して記載する
- 該当箇所が複数ページにわたる場合は `X〜Y ページ / 全 N ページ` と範囲で示す
- ページ数が取得できない場合は「ページ数不明」と記載する

**【PDF限定再検索オファー】**

参考URLにHTMLページとPDFが**混在**している場合、回答の最後に必ず以下のメッセージを表示する：

> 📎 回答にHTMLページとPDFの両方が含まれています。
> **「PDFに限定して再検索する」** と入力すると、PDFのみを対象に同じ質問で再検索します。

ユーザーが「PDFに限定して再検索する」または同趣旨の発言をした場合：
1. 直前の質問と同じキーワードで再度 `web_fetch` を実行
2. 取得URLが `.pdf` で終わるページのコンテンツのみを使用して回答を構成する
3. HTMLページの情報は使用しない
4. 参考URLはPDFのみを列挙する

## 利用例

### 例1: 通常の質問
**ユーザー:** 「雇い入れ当初週18時間の社員が残業で20時間を超えるようになった場合、雇用保険に加入すべきですか？」

→ キーワード抽出 → 検索 → 結果まとめ → 回答表示

**ユーザー:** 「対象サイトの一覧を見せて」

**Claude:**
> 現在の検索対象サイトは以下の通りです：
> 1. mhlw.go.jp（厚生労働省）

## スキルの使い方説明

ユーザーが「このスキルの使い方を知りたい」「このスキルをどうやって使うの？」など、スキルの使い方を尋ねた場合、以下の内容を説明する。

---

**このスキル（hr-search）でできること：**

人事・労務に関する行政・公的機関の公式サイトだけを情報源にして、あなたの質問に答えます。

**メリット：**
- 厚生労働省・年金機構・e-Govなど信頼性の高い公式情報のみを参照するため、民間サイトや個人ブログなどの不正確な情報に惑わされない
- 雇用保険・社会保険・労働基準・育休・有給・就業規則など、人事・労務の実務に関する質問を自然な文章で聞ける
- 複数の省庁サイトをまとめて調べ、結論を分かりやすく整理して回答する

**使い方：**
普通に質問するだけでOKです。例えば：
- 「育児休業給付金はいつまでもらえますか？」
- 「パートの社会保険加入条件を教えてください」
- 「時間外労働の上限規制はどうなっていますか？」
- 「試用期間中の解雇は認められますか？」

**対象サイトの管理：**
- 「対象サイトの一覧を見せて」→ 現在の検索対象サイト一覧を表示
- 「〇〇.go.jp を追加して」→ サイトを追加
- 「〇〇.go.jp を削除して」→ サイトを除外
- 「デフォルトに戻して」→ 初期設定にリセット

---

## 注意事項

- 検索結果が行政サイト以外のページを含む場合、その情報は無視して行政サイトの情報のみを使う（フェーズBで範囲拡大した場合を除く）
- 質問の内容が複数の省庁にまたがる場合（例：雇用保険＋年金）、複数のサイトを横断して検索・統合する
- 法改正がある分野（育児休業、最低賃金など）は情報の日付に注意し、最新情報を優先する
- **回答の末尾に「最終的には所管機関にご確認ください」「○○機関にお問い合わせください」など、確認・問い合わせを促す定型メッセージを付加しない**
- **回答の末尾に「このスキルは行政サイトを対象にしています」「行政公式サイトを検索しました」など、スキルの動作説明や情報源の種類を説明するメッセージを付加しない**
